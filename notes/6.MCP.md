## MCP (Model Context Protocol) – Short Summary

* **MCP** is an **open-source standard** that lets AI models connect to external tools and data sources.
* It solves the problem of building **different integrations** for every AI model and application.
* The main idea is **"Build once, use everywhere."**
* MCP is similar to **REST APIs** for web applications—it standardizes communication.
* An **MCP server** acts as a bridge between AI applications and services like GitHub, Google Drive, Asana, Salesforce, databases, etc.
* It allows AI to **read and write data** using natural language with minimal code.
* Anyone can build MCP servers, and they can be reused across multiple AI applications.
* MCP is **model-agnostic**, meaning it works with different LLMs like Claude, GPT, and Gemini.

### Key Takeaway

**MCP standardizes how AI applications access external tools and data, making integrations reusable, simpler, and scalable.**

## MCP Architecture – Short Summary

* MCP follows a **client-server architecture**.
* A **Host** (e.g., Claude Desktop, Cursor) contains one or more **MCP Clients**.
* Each **MCP Client** maintains a **1:1 connection** with an **MCP Server**.
* MCP Servers expose three main primitives:
<img width="865" height="357" alt="image" src="https://github.com/user-attachments/assets/7956361e-f7dd-47b2-80ec-923dd37dd17e" />
<img width="822" height="447" alt="image" src="https://github.com/user-attachments/assets/83248f7b-7a47-4349-b0aa-e801e976bda7" />
  * **Tools** – Functions that perform actions (e.g., query DB, update records).
  * **Resources** – Read-only data or context (e.g., files, PDFs, database records).
  * **Prompts** – Predefined prompt templates to reduce prompt engineering.
* Clients discover and use tools/resources, while servers provide them.
* MCP communication follows:

<img width="906" height="390" alt="image" src="https://github.com/user-attachments/assets/a92ede1b-36c4-48c5-a625-fb8e838f363d" />
  1. **Initialize connection**
  2. **Exchange requests, responses, and notifications**
  3. **Terminate connection**
* MCP supports different **transports**:
<img width="906" height="390" alt="image" src="https://github.com/user-attachments/assets/aa6a9a6b-71ad-4b29-9233-da5357dceb83" />

  * **Standard I/O (stdio)** – For local servers.
  * **HTTP + Server-Sent Events (SSE)** – For remote stateful servers.
  * **Streamable HTTP** – Modern transport supporting both **stateful and stateless** communication.
* The Python MCP SDK makes it easy to create servers by defining tools, resources, and prompts with decorators.

### Key Takeaway

**MCP uses a client-server architecture where hosts connect to MCP servers to access tools, resources, and prompt templates, enabling AI applications to interact with external systems in a standardized way.**

There are two ways for creating an MCP server:
- **low-level implementation**: in this approach, you directly define and handle the various types of requests (`ListToolsRequest` and `CallToolRequest`). This approach allows you to customize every aspect of your server.
- **high-level implementation using `FastMCP`**: `FastMCP` is a high-level interface that makes building MCP servers faster and simpler. In this approach, you just focus on defining the tools as functions, and`FastMCP` handles all the protocol details.
  
You will use in this lesson `FastMCP`. If you'd like to learn more about the low-level approach, you can check out the resources at the end of this notebook.

To create your MCP server using `FastMCP`, you will initialize a `FastMCP` server labeled as `mcp` and decorating the functions with `@mcp.tool()`. `FastMCP` automatically generates the necessary MCP schema based on type hints and docstrings.


**Note**: The magic function `%%writefile mcp_project/research_server.py` will not execute the code but it will save the content of the cell to the server file `research_server.py` in the directory: `mcp_project`. If you remove the magic function and run the cell, the code won't run in Jupyter notebook. You will run the server from the terminal in the next section. 


```python
%%writefile mcp_project/research_server.py

import arxiv
import json
import os
from typing import List
from mcp.server.fastmcp import FastMCP


PAPER_DIR = "papers"

# Initialize FastMCP server
mcp = FastMCP("research")

@mcp.tool()
def search_papers(topic: str, max_results: int = 5) -> List[str]:
    """
    Search for papers on arXiv based on a topic and store their information.
    
    Args:
        topic: The topic to search for
        max_results: Maximum number of results to retrieve (default: 5)
        
    Returns:
        List of paper IDs found in the search
    """
    
    # Use arxiv to find the papers 
    client = arxiv.Client()

    # Search for the most relevant articles matching the queried topic
    search = arxiv.Search(
        query = topic,
        max_results = max_results,
        sort_by = arxiv.SortCriterion.Relevance
    )

    papers = client.results(search)
    
    # Create directory for this topic
    path = os.path.join(PAPER_DIR, topic.lower().replace(" ", "_"))
    os.makedirs(path, exist_ok=True)
    
    file_path = os.path.join(path, "papers_info.json")

    # Try to load existing papers info
    try:
        with open(file_path, "r") as json_file:
            papers_info = json.load(json_file)
    except (FileNotFoundError, json.JSONDecodeError):
        papers_info = {}

    # Process each paper and add to papers_info  
    paper_ids = []
    for paper in papers:
        paper_ids.append(paper.get_short_id())
        paper_info = {
            'title': paper.title,
            'authors': [author.name for author in paper.authors],
            'summary': paper.summary,
            'pdf_url': paper.pdf_url,
            'published': str(paper.published.date())
        }
        papers_info[paper.get_short_id()] = paper_info
    
    # Save updated papers_info to json file
    with open(file_path, "w") as json_file:
        json.dump(papers_info, json_file, indent=2)
    
    print(f"Results are saved in: {file_path}")
    
    return paper_ids

@mcp.tool()
def extract_info(paper_id: str) -> str:
    """
    Search for information about a specific paper across all topic directories.
    
    Args:
        paper_id: The ID of the paper to look for
        
    Returns:
        JSON string with paper information if found, error message if not found
    """
 
    for item in os.listdir(PAPER_DIR):
        item_path = os.path.join(PAPER_DIR, item)
        if os.path.isdir(item_path):
            file_path = os.path.join(item_path, "papers_info.json")
            if os.path.isfile(file_path):
                try:
                    with open(file_path, "r") as json_file:
                        papers_info = json.load(json_file)
                        if paper_id in papers_info:
                            return json.dumps(papers_info[paper_id], indent=2)
                except (FileNotFoundError, json.JSONDecodeError) as e:
                    print(f"Error reading {file_path}: {str(e)}")
                    continue
    
    return f"There's no saved information related to paper {paper_id}."



if __name__ == "__main__":
    # Initialize and run the server
    mcp.run(transport='stdio')
```
