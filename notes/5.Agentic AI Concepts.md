## Module 2: Reflection Design Pattern  

### Reflection Design Pattern

The **Reflection Design Pattern** is an AI approach where a model generates an initial output, reviews it, and then produces an improved version. It can use the same or a different model for the reflection step, depending on the task. Reflection is most effective when combined with **external feedback** (such as test results, error messages, or user feedback), enabling the AI to identify and fix issues before generating the final response.

**Key Points:**

* Generate → Review → Improve workflow.
* Uses the same or a different model for reflection.
* Improves writing, coding, and reasoning tasks.
* External feedback makes reflection much more effective.
* Produces better results than a single-pass (zero-shot) approach, though it doesn't guarantee perfect accuracy.
